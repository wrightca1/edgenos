# IEEE 802.3 10G Ethernet & MDIO Reference

Quick reference for AS5610-52X SFP+ port debugging.

## Clause 22 MDIO (MII Management)

Classic MII management interface for PHY register access.

### Frame Format (single frame)
```
PRE(32) | ST(01) | OP(2) | PHYAD(5) | REGAD(5) | TA(2) | DATA(16)
```
- **ST**: Start of frame = `01`
- **OP**: `01` = write, `10` = read
- **PHYAD**: PHY address (0-31)
- **REGAD**: Register address (0-31) - only 32 registers!
- **TA**: Turnaround (2 bits)

### Key Registers (Clause 22)
| Reg | Name | Key Bits |
|-----|------|----------|
| 0 | Control | bit15=reset, bit12=autoneg_enable, bit13=speed_MSB, bit6=speed_LSB, bit8=duplex |
| 1 | Status | bit2=link_status, bit5=autoneg_complete, bit3=autoneg_ability |
| 2-3 | PHY Identifier | OUI |
| 4 | Autoneg Advertisement | |
| 5 | Autoneg Link Partner Ability | |

### Limitation
- 32 registers x 32 PHYs - insufficient for 10G PHYs
- WARPcore uses Clause 22 with indirect addressing (AER register for lane/block selection)

## Clause 45 MDIO (Extended Management)

Extended interface for 10G+ PHYs. Two-frame access (address frame + data frame).

### Frame Format
```
Address frame: PRE(32) | ST(00) | OP(00) | PRTAD(5) | DEVAD(5) | TA(2) | ADDR(16)
Data frame:    PRE(32) | ST(00) | OP(01/10/11) | PRTAD(5) | DEVAD(5) | TA(2) | DATA(16)
```
- **ST**: `00` for Clause 45
- **OP**: `00`=address, `01`=write, `10`=read_inc_addr, `11`=read
- **PRTAD**: Port address (0-31)
- **DEVAD**: MMD device type (see below)
- **ADDR**: 16-bit register address (65536 registers per MMD!)

### MMD Device Types (DEVAD)
| DEVAD | Device | Purpose |
|-------|--------|---------|
| 1 | PMA/PMD | Physical Medium Attachment/Dependent - SerDes control |
| 2 | WIS | WAN Interface Sublayer (SONET) |
| 3 | PCS | Physical Coding Sublayer - 64B/66B encoding |
| 4 | PHY XS | PHY Extender Sublayer (XGXS) |
| 5 | DTE XS | DTE Extender Sublayer |
| 7 | Auto-negotiation | CL73 auto-negotiation |
| 30 | Vendor Specific 1 | |
| 31 | Vendor Specific 2 | |

### Key PMA/PMD Registers (DEVAD=1)
| Register | Name | Description |
|----------|------|-------------|
| 1.0 | PMA/PMD Control 1 | bit15=reset, bit11=low_power, bit0=PMA loopback |
| 1.1 | PMA/PMD Status 1 | bit2=link_status, bit1=low_power_ability |
| 1.2-1.3 | Device Identifier | |
| 1.4 | Speed Ability | bit0=10G capable |
| 1.5-1.6 | Devices in Package | |
| 1.7 | PMA/PMD Control 2 | PMA type select (10GBASE-SR/LR/ER/etc.) |
| 1.8 | PMA/PMD Status 2 | bit10=local_fault, bit13=tx_fault, bit14=rx_fault |
| **1.9** | **PMD Transmit Disable** | **Per-lane TX disable! bit0=global, bits 1-4=lane 0-3** |
| 1.10 | PMD Signal Detect | bit0=global, bits 1-4=lane 0-3 signal present |
| 1.11 | PMD 10G Extended Ability | |

### Key PCS Registers (DEVAD=3)
| Register | Name | Description |
|----------|------|-------------|
| 3.0 | PCS Control 1 | bit15=reset, bit0=PCS loopback |
| 3.1 | PCS Status 1 | bit2=PCS link status, bit12=rx_link_status |

## 10GBASE-R PCS (Clause 49)

The PCS for 10G uses 64B/66B encoding:

### Architecture
```
MAC → RS → PCS (64B/66B) → PMA → PMD → Medium
                 ↕
              XGMII (internal) or XFI/SFI (serial)
```

### 64B/66B Encoding
- 64 data bits + 2-bit sync header = 66-bit block
- Sync header: `01` = data block, `10` = control block
- Self-synchronizing scrambler (polynomial x^58 + x^39 + 1)
- Line rate: 10.3125 Gbps (10G data + 3.125% overhead)

### Block Types
- **D block** (sync=01): 8 bytes of data
- **C block** (sync=10): Control characters (idle, start, terminate, error, ordered set)
- Idle pattern: continuous stream of control blocks with idle characters

### XFI vs SFI
- **XFI**: 10G serial electrical interface between MAC/ASIC and optical module
  - Direct connection, no CDR/retimer needed for short traces
  - Used in SFP+ direct attach
- **SFI**: 10G serial interface requiring SerDes with CDR
  - Used between ASIC and external PHY
  - Longer trace lengths, needs equalization

For AS5610-52X: WARPcore outputs XFI → DS100DF410 retimer (CDR + EQ) → SFP+ cage

## WARPcore Internal PHY (Broadcom)

The WARPcore is a quad-lane SerDes integrated into the BCM56846. It uses Clause 22 MDIO with extended indirect addressing (AER register).

### MDIO Access via Clause 22
- Internal MIIM bus (IBUS) for WARPcore access
- AER (Address Extension Register) at 0x1F for block/lane selection
- Each WARPcore has 4 lanes, accessed via PHY instance

### Critical WARPcore Registers
- **XGXSCONTROL**: PLL sequencer start, mode select
- **MISC1**: Force PLL mode, force speed
- **MISC3**: LANEDISABLE (per-lane disable!)
- **RXLNSWAP1**: RX lane remapping
- **TXLNSWAP1**: TX lane remapping
- **FIRMWARE_MODE**: Per-lane firmware mode (SFP_DAC, etc.)
- **PMD TX DISABLE**: Global + per-lane TX disable bits (bit 11-15 of a control register)

### PMD Transmit Disable (IEEE 802.3 register 1.9)
```
Bit 0: Global PMD transmit disable
Bit 1: PMD transmit disable lane 0
Bit 2: PMD transmit disable lane 1
Bit 3: PMD transmit disable lane 2
Bit 4: PMD transmit disable lane 3
```
**This register can independently disable TX on each WARPcore lane.**
